## üê¶ Content Strategy

### Text-First, Real-Time Platform
Twitter is about **immediate, concise communication**. Every tweet must:
- Be concise and punchy (280 characters max)
- Provide immediate value or insight
- Be timely and relevant to current events
- Invite engagement and conversation

### Content Pillars (4 Primary Types)

#### 1. **Breaking News & Updates** (30%)
**Purpose**: Announce timely business updates, launches, wins  
**Format**: Short announcement format  
**Length**: 150-200 characters (leaves room for retweets with comments)  

**Template**:
```
[Emoji] [What happened] [Why it matters] #Hashtag

Example:
"üöÄ Just launched our new [product] for [client/industry].
First to market with [key feature].
#ProductLaunch #Innovation"
```

#### 2. **Quick Tips & Insights** (40%)
**Purpose**: Share knowledge, lessons learned, industry observations  
**Format**: Actionable tip or thought-provoking statement  
**Length**: 200-250 characters  

**Template**:
```
[Observation/Tip] [Why it matters] [Optional CTA]

Example:
"Pro tip: [Specific action].
Result: [Measurable benefit].
What's worked for you?
#[Industry]Tip #BestPractices"
```

#### 3. **Thread-Based Deep Dives** (20%)
**Purpose**: Detailed explanations, case studies, processes  
**Format**: Tweet thread (3-10 tweets)  
**Structure**: Hook ‚Üí Details ‚Üí Conclusion  

**Template**:
```
Tweet 1 (Hook):
"üßµ How we [achieved result] in [timeframe].
A thread on [topic]."

Tweet 2-9 (Details):
"1/ [Step or point]"
"2/ [Step or point]"
...

Tweet 10 (Conclusion):
"That's it! Key takeaway: [Main point].
Questions? Drop them below üëá"
```

#### 4. **Engagement Posts** (10%)
**Purpose**: Foster community, ask questions, polls  
**Format**: Question or poll  
**Length**: 100-150 characters  

**Template**:
```
[Question relevant to audience]
What's your experience?

Example:
"Quick poll for [industry] folks:
What's your biggest challenge right now?
A) [Option]
B) [Option]
C) [Option]
Comment below üëá"
```

## ‚úçÔ∏è Writing Best Practices

### Character Limits
- **Maximum**: 280 characters
- **Optimal**: 200-250 characters (easier to retweet with comments)
- **Absolute Minimum**: 100 characters (too short looks low-effort)

### Structure for Clarity
Use line breaks for readability:
```
Good:
"Pro tip for [audience]:
[Action].
Result: [Benefit].
#Hashtag"

Bad (all one line):
"Pro tip for [audience]: [Action]. Result: [Benefit]. #Hashtag"
```

### Emoji Usage
- **Amount**: 1-2 per tweet (not excessive)
- **Placement**: Opening (attention) or closing (emphasis)
- **Common**: üí° (ideas), üöÄ (launch), ‚úÖ (completed), üßµ (thread), üìà (growth), ‚ö° (speed/power)

### Tone & Voice
- **Professional but conversational**: Not corporate jargon
- **Confident but humble**: Share wins, acknowledge challenges
- **Helpful**: Always provide value
- **Authentic**: Real experiences, not polished PR speak

### Call-to-Action (CTA)
Every tweet should invite engagement:
- "What's your experience?"
- "Thoughts?"
- "Drop your [question/answer] below üëá"
- "Agree/Disagree?"
- "Tag someone who needs this."

## #Ô∏è‚É£ Hashtag Strategy

### Amount
- **Optimal**: 1-2 hashtags per tweet
- **Maximum**: 3 hashtags (performance drops after 2)
- **Threads**: Only on first and last tweet

### Types
1. **Industry Tag** (always include): #[YourIndustry] #[YourNiche]
2. **Topic Tag** (content-specific): #Productivity #Marketing #Leadership

### Placement
- End of tweet (not mid-sentence)
- On separate line for readability

### Research
- Check trending hashtags daily (Twitter Explore)
- Use tags with 10K-100K tweets (not too broad, not too niche)
- Avoid #FollowFriday #RT4RT and spam tags

## üßµ Thread Best Practices

### When to Use Threads
- Deep dive into process or case study
- Step-by-step tutorials
- Multiple points that don't fit in one tweet
- Storytelling (beginning, middle, end)

### Thread Structure

**Tweet 1 (Hook)**:
- Must stand alone (many won't read further)
- Tease value: "How we [result] - a thread üßµ"
- Clear benefit statement

**Tweets 2-9 (Body)**:
- Number each: "1/", "2/", etc.
- One point per tweet
- Use visuals if available (screenshots, diagrams)
- Keep each tweet readable standalone

**Last Tweet (CTA)**:
- Summarize key takeaway
- CTA: "Follow for more [topic]"
- Invite discussion: "Questions? Reply below."

### Thread Limits
- **Optimal**: 5-7 tweets (most people read this far)
- **Maximum**: 10 tweets (attention drops after this)
- **Minimum**: 3 tweets (otherwise just write one tweet)

## üîí Human-in-the-Loop (HITL) Approval

### Auto-Approve (‚úÖ No HITL)

- Industry observations
- Completed project announcements (non-confidential)
- Weekly schedule tweets
- Engagement questions

### Requires CEO Approval (‚è∏Ô∏è HITL)
- Client-specific details (even with permission, double-check)
- Financial information (revenue, pricing)
- Controversial opinions
- Responses to negative press or criticism
- Partnership/collaboration announcements
- Anything mentioning competitors by name

### Approval Process
1. AI creates draft in `/Needs_Approval/`
2. Include:
   - Tweet text
   - Thread tweets (if applicable)
   - Reasoning (why this tweet, expected outcome)
   - Risk assessment (what could go wrong)
3. CEO reviews and renames file:
   - `.approved` = post immediately
   - `.rejected` = cancel, don't post
   - `.edit_[feedback]` = revise per instructions

## üìà Analytics & Performance Tracking

### Key Metrics (Check Weekly)
- **Impressions**: How many times tweets were seen
- **Engagements**: Likes + Retweets + Replies + Clicks
- **Engagement Rate**: Engagements / Impressions
  - Target: 2-3% (good), 3-5% (excellent), 5%+ (viral)
- **Profile Visits**: Clicks to your profile
- **Link Clicks**: If sharing content
- **Follower Growth**: Net new followers per week

### Access Analytics
- Twitter Analytics: analytics.twitter.com
- Or via Twitter API v2: `get_user_info()` method

### Weekly Review (Every Monday)
1. Review previous week's tweets
2. Identify top tweet (highest engagement rate)
3. Analyze why it performed well:
   - Time posted?
   - Topic resonance?
   - Format (thread vs. single tweet)?
4. Replicate successful patterns
5. Drop content types with low engagement

### Monthly Reporting
Include in CEO briefing:
- Total tweets published
- Average engagement rate
- Top 3 tweets (with metrics)
- Follower growth
- Notable mentions or conversations
- Recommendations for next month

## ‚ö†Ô∏è Risk Management & Content Guidelines

### Content to AVOID
‚ùå Never tweet:
- Confidential client information
- Unverified rumors or speculation
- Political or religious controversial topics
- Complaints about clients or partners
- Personal attacks or inflammatory language
- False or misleading claims

### Crisis Response

**If Tweet Backlash**:
1. Don't delete immediately (looks guilty)
2. Assess: Is criticism valid?
3. If you're wrong: Apologize clearly and publicly
4. If misunderstood: Clarify with additional tweet
5. Notify CEO via HITL if blowing up

**Example Apology**:
```
"We got this wrong. [What was wrong].
We're [corrective action]. Thanks for the feedback."
```

**If Account Compromised**:
1. Change password immediately
2. Revoke third-party app access
3. Enable two-factor authentication
4. Post: "Account was compromised. We've secured it. Please disregard recent tweets."

### Brand Safety
- Professional tone (even when casual)
- No profanity or offensive language
- Fact-check all statistics and claims
- Avoid clickbait or sensationalism
- Give credit when sharing others' insights

## üéØ Twitter-Specific Features

### Retweets
**When to Retweet**:
- Industry news relevant to your audience
- Client wins (with their permission)
- Team member achievements
- Valuable insights from thought leaders

**Retweet Types**:
1. **Quote Retweet** (preferred): Add your commentary
2. **Simple Retweet**: Only if content is perfect as-is

**Frequency**: 1-2 retweets per week (don't overdo it)

### Twitter Polls
**Use For**:
- Quick audience research
- Engagement boost
- Content ideation ("What topic next?")

**Best Practices**:
- 2-4 options (not too many)
- Clear, distinct choices
- Run for 24 hours (not 7 days)
- Share results after poll closes

**Example**:
```
"Quick poll for [industry] folks:
What's your biggest challenge right now?

üîµ Finding clients
üü¢ Pricing strategy
üü° Time management
üü† Something else (comment)

Voting closes in 24 hours."
```

### Twitter Spaces (Audio)
**Use For**:
- Q&A sessions
- Industry discussions
- Networking events

**Frequency**: Optional, not required for Gold tier  
**If Used**: Promote 24 hours in advance, recap key points in thread after

### Lists
**Use For**:
- Monitoring competitors
- Following industry leaders
- Organizing clients (private list)

**Don't**: Publicly list people in negative categories

## üõ†Ô∏è Technical Integration

### MCP Server Actions
```python
# Post single tweet
twitter_server.post_tweet(
    text="Tweet text here (max 280 characters) #Hashtag",
    dry_run=False
)

# Post thread
twitter_server.post_thread(
    tweets=[
        "üßµ Thread hook (standalone value)",
        "1/ First point with details",
        "2/ Second point with details",
        "That's it! Key takeaway here."
    ],
    dry_run=False
)

# Reply to tweet
twitter_server.reply_to_tweet(
    tweet_id="1234567890",
    text="Thanks for sharing! Here's our take...",
    dry_run=False
)

# Get recent tweets
twitter_server.get_tweets(limit=10)

# Get mentions
twitter_server.get_mentions(limit=10)

# Get user info
twitter_server.get_user_info()
```

### Task Queue Integration
Twitter watcher creates tasks in `task_queue/inbox/` with:
- `task_type: "twitter_action"`
- `trigger: "breaking_news" | "quick_win" | "insight" | "weekly_schedule"`
- `content`: Relevant context (update text, win description, insight)
- `instructions`: AI guidance from `_get_instructions()`

Orchestrator claims task, follows instructions, executes MCP action, logs result.

### Rate Limiting
Watcher enforces rate limits to avoid over-posting:
- **Breaking News**: Max 3/day
- **Quick Wins**: Max 5/day
- **Insights**: Max 3/day
- **Weekly Schedule**: Max 1/day

Tracked in `task_queue/.twitter_recent.json` (24-hour rolling window).
